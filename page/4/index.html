git多用户设置ssh key

发表于 2016-07-28   |  

  • 使用不同git（gitlab/github）时，可以使用同一个key，但在github上不用账号使用同一个key时，就会提示不能使用。这时我们就需要进行另外的设置，先生成key:

    // 切换到ssh的目录
cd ~/.ssh
// 执行密钥生成命令
ssh-keygen -t rsa -C "youremail@xxx.com"
// 输入文件名，不输入会使用默认设置，会覆盖以前的
Enter file in which to save the key (/Users/xxx/.ssh/id_rsa):test
// 可以直接回车然后就会在当前目录生成两个文件，test和test.pub。
Enter passphrase (empty for no passphrase):
// 获取密钥，添加到git账户中
cat test.pub
// 在.ssh/下创建config文件
vim config
// 更具需求酌情更改
Host github.com  
  HostName github.com  
  PreferredAuthentications publickey
  IdentityFile ~/.ssh/id_rsa  
Host my.github.com
  HostName github.com
  PreferredAuthentications publickey  
  IdentityFile ~/.ssh/test

  • 最后第二个账户，使用到github.com的地方，替换成my.github.com

Ubuntu 安装 Hexo 记录

发表于 2016-07-26   |  

安装官网安装步骤安装，往往会出现问题。

  • ###安装git:
    $ sudo apt-get install git-core目前没出现过问题。

  • ###安装 Node.js
    官方推荐用nvm安装。Ubuntu下安装，看网友提示说要安装nodejs-legacy，安装方法：$ sudo apt-get install nodejs-legacy，详情https://github.com/hexojs/hexo/issues/580。

    ####安装nvm
    cURL:
    $ curl https://raw.github.com/creationix/nvm/master/install.sh | sh
    Wget:
    $ wget -qO- https://raw.github.com/creationix/nvm/master/install.sh | sh
    安装nvm时，执行这阳光命令分别都出现了问题，但多执行几次后安装成功了。
    其中遇到的一个问题就是输入nvm提示没有安装nvm，然后再次安装提示已经安装，疑似某个配置出现问题，没有找到该命令。查询资料得知，在~/.bashrc 文件里配置nvm，发现配置正确。
    export NVM_DIR="/root/.nvm" [ -s "$NVM_DIR/nvm.sh" ] && . "$NVM_DIR/nvm.sh" # This loads nvm
    最后输入source .bashrc奇迹般的好了。

  • ###安装hexo
    $ npm install hexo-cli -g，由于之前设置过系统代理，看错误log是不能连接网络，取消代理，依然不行，重启电脑好了。

  • 使用hexo

最后在浏览器输入localhost:4000就好了。
